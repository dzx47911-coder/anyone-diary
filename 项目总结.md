# 小董日记 - 项目总结

## 项目定位

小董日记是一款面向年轻用户的云端手账风日记 Web 应用。用户可以在任何设备的浏览器上记录日记，选择心情标签，生成手账风格的卡片，并通过日历、时间轴、总结等多种视图回顾自己的记录。

核心卖点：手账风视觉体验 + 云端多设备同步 + 零安装浏览器直接使用。

## 技术架构

```
用户浏览器 ←→ Vercel（前端托管）←→ Render（后端API）←→ Neon（PostgreSQL数据库）
```

| 层级 | 技术 | 说明 |
|------|------|------|
| 前端 | React 18 + Vite | 组件化开发，手账风CSS |
| 后端 | Node.js + Express | RESTful API，JWT认证 |
| 数据库 | Neon PostgreSQL | 云端关系型数据库 |
| 前端托管 | Vercel | 自动部署，自定义域名 doiandoian.xyz |
| 后端托管 | Render | 自动部署，免费额度 |
| 代码仓库 | GitHub | 推送即触发自动部署 |

## 用户流程

```
注册/登录 → 给日记本命名 → 首页
                                ├→ 写日记（选日期、心情标签、写内容）→ 生成手账卡片 → 保存
                                ├→ 日历视图（按月查看，点击日期看日记）
                                ├→ 时间轴（按时间倒序浏览所有日记）
                                ├→ 总结页（年度/月度统计：篇数、字数、活跃月份）
                                └→ 退出登录（二次确认）
```

## API 设计

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | /api/auth/register | 注册（用户名+密码） |
| POST | /api/auth/login | 登录，返回JWT |
| GET | /api/diaries | 获取当前用户所有日记 |
| GET | /api/diaries/:date | 获取指定日期日记 |
| POST | /api/diaries | 创建/更新日记（同日期覆盖） |
| DELETE | /api/diaries/:id | 删除日记 |

## 数据模型

```sql
-- 用户表
users: id, username, password_hash, avatar, created_at

-- 日记表
diaries: id, user_id, date, moods(JSONB), mood_labels(JSONB), custom_moods(JSONB), content, created_at, updated_at
-- 每个用户每天只能有一篇日记（user_id + date 唯一索引）
```

## 设计风格

- 配色：粉色(#FFB6C1) + 米白(#FFF8F0) + 浅粉(#FFE4E1)
- 字体：站酷快乐体（标题）、Noto Sans SC（正文）
- 风格：黏土风/手账风，圆角卡片、柔和阴影、胶带装饰、浮动花瓣动画
- 交互：时间轴滚动渐入动画、日历点击弹窗、卡片悬停效果

## 已做的关键决策

| 决策 | 选择 | 原因 |
|------|------|------|
| 前端框架 | React + Vite | 组件化开发，热更新快 |
| 后端托管 | Render（弃用Railway） | Railway端口问题多，Render更稳定 |
| 前端托管 | Vercel（弃用Netlify） | Netlify被墙，Vercel国内可访问 |
| 数据库 | Neon PostgreSQL（弃用sql.js） | sql.js无法持久化，Neon免费云端 |
| 认证方式 | 用户名+密码（弃用邮箱） | 简化注册流程，降低门槛 |
| 数据同步 | 云端API | 支持多设备，数据不丢失 |
| 日记唯一性 | 每用户每天一篇 | 简化逻辑，同日期自动覆盖 |

## 已知问题与技术债

1. CORS 设置为 `origin: '*'`，生产环境应限制为具体域名
2. JWT密钥有硬编码兜底值，应在Render环境变量中设置
3. 前端API_URL硬编码在两个文件中（App.jsx、AuthPage.jsx），应统一
4. 无分页机制，日记量大时性能会下降
5. 日记内容明文存储，无加密
6. 无速率限制，存在暴力破解风险
