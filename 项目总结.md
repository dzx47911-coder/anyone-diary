# Anyone diary - 项目总结

## 项目定位

Anyone diary 是一款面向年轻用户的云端手账风日记 Web 应用。用户可以在任何设备的浏览器上记录日记，选择心情标签和马卡龙配色，生成手账风格的卡片，并通过日历、时间轴、搜索、总结等多种视图回顾自己的记录。

核心卖点：手账风视觉体验 + 云端多设备同步 + 零安装浏览器直接使用。

## 技术架构

```
用户浏览器 ←→ Vercel（前端托管）←→ Render（后端API）←→ Neon（PostgreSQL数据库）
```

| 层级 | 技术 | 说明 |
|------|------|------|
| 前端 | React 18 + Vite | 组件化开发，手账风CSS |
| 后端 | Node.js + Express | RESTful API，JWT认证 |
| 数据库 | Neon PostgreSQL | 云端关系型数据库，JSONB存储心情数据 |
| 前端托管 | Vercel | 自动部署，自定义域名 doiandoian.xyz，SPA重写规则 |
| 后端托管 | Render | 自动部署，免费额度 |
| 代码仓库 | GitHub | 推送即触发自动部署 |
| 图片导出 | html2canvas | 卡片截图导出为PNG |

## 用户流程

```
注册 → 给日记本命名 → 首页
登录 → 直接进入首页（保留上次名称）

首页
├→ 写日记（选日期、写内容、选心情标签、选卡片颜色）
│   → 生成卡片预览（可换颜色、返回编辑）
│   → 保存 → 跳转日历
├→ 日历（点击有日记的日期查看，无日记弹出"立即写一篇"）
├→ 时间轴（同日多篇悬停展开，点击单篇弹窗查看）
├→ 搜索（关键词+心情+日期范围筛选，点击结果弹窗查看）
├→ 总结（年度/月度统计：篇数、字数、活跃月份）
└→ 退出登录（二次确认）
```

## API 设计

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | /api/auth/register | 注册（用户名+密码） |
| POST | /api/auth/login | 登录，返回JWT |
| GET | /api/diaries | 获取当前用户所有日记 |
| GET | /api/diaries/:date | 获取指定日期日记 |
| POST | /api/diaries | 创建日记（同日可多篇，不覆盖） |
| DELETE | /api/diaries/:id | 删除单篇日记 |

## 数据模型

```sql
-- 用户表
users: id, username, password_hash, avatar, created_at

-- 日记表（同日可多篇，无唯一索引）
diaries: id, user_id, date, moods(JSONB), mood_labels(JSONB),
         custom_moods(JSONB), card_style(卡片颜色), content,
         created_at, updated_at
```

## 设计风格

- 品牌名：Anyone diary
- 配色：马卡龙8色系（樱花粉、薄荷绿、奶油黄、薰衣草紫、天空蓝、蜜桃橙、抹茶绿、云朵白）
- 字体：ZCOOL XiaoWei（全局统一）、ZCOOL KuaiLe（日记内容手写风）
- 风格：黏土风/手账风，圆角卡片、柔和阴影、胶带装饰
- 图标：统一手绘粉色系SVG图标（首页、日历、编辑、搜索、相机等）
- 动画：漂浮物鼠标躲避、时间轴滚动渐入、同日卡片悬停展开
- 导出：同色系边框PNG图片

## 核心功能清单

| 功能 | 说明 |
|------|------|
| 用户注册/登录 | 用户名+密码，JWT认证，30天有效 |
| 日记本命名 | 新注册触发，登录保留上次名称，可随时修改 |
| 写日记 | 日期+内容+心情标签（可选）+卡片颜色 |
| 生成卡片预览 | 预览后可换颜色、返回编辑，保存才最终提交 |
| 每天多篇日记 | 同日不覆盖，每篇独立保存和删除 |
| 日历视图 | 手写勾标记有日记的日期，无日记弹"立即写一篇" |
| 时间轴 | 同日多篇叠放悬停展开，点击单篇弹窗查看 |
| 搜索 | 关键词+心情标签+日期范围，摘要列表+点击弹窗 |
| 总结 | 年度概览+月度详情（篇数、字数、活跃月份） |
| 导出图片 | 二次确认，同色系边框PNG导出 |
| 漂浮装饰 | 首页/时间轴浮动emoji，鼠标靠近自动躲避 |
| 退出登录 | 二次确认弹窗 |

## 已做的关键决策

| 决策 | 选择 | 原因 |
|------|------|------|
| 前端框架 | React + Vite | 组件化开发，热更新快 |
| 后端托管 | Render（弃用Railway） | Railway端口问题多，Render更稳定 |
| 前端托管 | Vercel（弃用Netlify） | Netlify被墙，Vercel国内可访问 |
| 数据库 | Neon PostgreSQL（弃用sql.js） | sql.js无法持久化，Neon免费云端 |
| 认证方式 | 用户名+密码（弃用邮箱） | 简化注册流程，降低门槛 |
| 日记存储 | 每天可多篇（弃用覆盖模式） | 用户需求，每篇独立 |
| 卡片风格 | 马卡龙颜色（弃用多模板） | 更简洁，用户体验更好 |
| 写日记流程 | 预览→保存两步 | 防止误操作，可返回编辑 |
| SPA路由 | vercel.json重写规则 | 解决刷新404问题 |

## 已知问题与技术债

1. CORS 设置为 `origin: '*'`，生产环境应限制为具体域名
2. JWT密钥有硬编码兜底值，应在Render环境变量中设置
3. 前端API_URL硬编码在两个文件中（App.jsx、AuthPage.jsx），应统一
4. 无分页机制，日记量大时性能会下降
5. 日记内容明文存储，无加密
6. 无速率限制，存在暴力破解风险
7. 忘记密码功能未实现（需要邮件服务）
8. 头像上传功能未实现（计划用ImgBB）
